# GestureClip 产品目标

## 北极星目标

把 GestureClip 做成 **小白也敢用、办公党每天离不开的 Windows 本地办公加速器**。

它不是“功能堆叠工具”，而是一个让用户在复制、粘贴、找回内容、鼠标手势、快捷动作和办公状态提醒上明显省时间、少出错、敢升级的本地软件。

## 一句话定位

> GestureClip：不登录、不上传、不卡手的本地剪贴板历史 + 鼠标手势办公加速器。

## 目标用户

- 每天高频复制粘贴的办公用户。
- 客服、运营、写作、数据整理、开发、内容编辑人员。
- 想要剪贴板历史，但不希望内容上云的人。
- 想用鼠标手势提升效率，但不想学习复杂配置的人。
- 不懂技术、只想“下载、解压、双击、能用”的小白用户。

## v1.0 完成标准

### 1. 第一次使用闭环

- 用户下载 zip 后，能看懂怎么解压和启动。
- 首次打开能看到 30 秒内理解的引导：快捷键、剪贴板、手势、数据本地保存。
- 默认功能安全可用，不需要先配置一堆参数。
- 缺权限、快捷键冲突、手势不可用时，用人话解释，不暴露吓人的底层异常。

### 2. 核心动作稳定

- 剪贴板历史打开、搜索、复制、粘贴不卡死。
- 文本和图片历史不会因为长文本、大图、异常剪贴板格式拖垮 UI。
- 鼠标手势识别稳定，可配置、可删除、避免误触。
- 设置页任何点击都不能卡死；所有危险操作必须确认。

### 3. 本地优先可信任

- 明确告诉用户：不登录、不上传、不云同步。
- 能看到本地数据目录。
- 能一键导出诊断包，便于反馈问题。
- 清空/删除操作说明“只影响本机历史，不影响当前系统剪贴板”。

### 4. 更新发布可信任

- 每个用户可见功能修复都要 bump 版本并发 GitHub Release。
- README 永远指向 Latest Release。
- Release 包含 zip 和 SHA256SUMS。
- 更新说明必须用小白能懂的话写清楚：修了什么、怎么用、会不会影响本地数据。

### 5. 小白友好 UX

- 按钮文案直接说明结果，比如“复制到系统剪贴板”“删除这个手势”。
- 危险按钮不要藏在远处，也不要误触即执行。
- 选中态明显，不依赖系统默认黑块高亮。
- 操作失败时提供下一步建议，而不是只给错误码。

## 不做什么

- 不做账号系统。
- 不做云同步。
- 不上传剪贴板内容。
- 不为了炫技加入复杂配置。
- 不把测试版功能包装成“稳定正式版”。

## 当前优先级

1. **稳定性优先**：任何卡死、崩溃、明显卡顿都优先于新功能。
2. **小白闭环优先**：安装、首次使用、更新、诊断、删除确认必须清楚。
3. **核心效率优先**：剪贴板历史、搜索、复制粘贴、鼠标手势优先。
4. **发布闭环优先**：用户可见改动必须发新版，而不是只提交代码。

## 每次优化的验收清单

- 是否解决一个真实小白痛点？
- 是否有回归测试或至少源码级 UI 合约测试？
- 是否跑过 `git diff --check`？
- 是否跑过 `dotnet test ./GestureClip.sln`？
- 如果用户可见，是否更新版本号、CHANGELOG、README、Release notes？
- 是否生成并校验发布包？
- 是否确认 GitHub Release 是 Latest？
